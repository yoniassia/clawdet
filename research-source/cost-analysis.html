⚡ Key Insight: Hot/Cold Container Architecture

At 50K+ scale, keeping all containers in RAM is impractical. The recommended approach is hot/cold scheduling: active containers run in RAM, idle containers are suspended to disk and restored on demand (~2-5s cold start). This reduces RAM requirements by 80-95% at the 500K tier.

🏆 5K Recommendation: Hetzner Dedicated + K3s

8× AX162 servers at €150/mo each. K3s provides lightweight orchestration with auto-restart, rolling updates, health checks, and namespace isolation. Best price-performance by far — 3× cheaper than next option, 14× cheaper than cloud providers.

  • Why not full K8s: Overkill for identical containers; K3s is simpler with same core features
  • Why not cloud: 10-30× more expensive for same compute

🔶 Tier 2: 50,000 Agents

2,500 concurrent (5%) · Hot/cold model essential · 3,625 vCPU peak

⚠️ Architecture Shift Required at 50K

At 50K, keeping all containers running (20TB RAM) is cost-prohibitive on any provider. A hot/cold scheduling layer is mandatory: active containers in RAM (~1.4TB), idle containers suspended to NVMe (~10TB disk). This requires a custom scheduler or KEDA-based auto-scaling. Cold start: 2-5 seconds.

🏆 50K Recommendation: Hetzner Dedicated + K8s (Warm Oversubscription)

€12,000/mo with 80 AX162 nodes. At 50K, K3s is replaced with full K8s for better multi-node management, network policies, and RBAC. Oversubscribe memory with swap — idle containers use ~50MB resident + swap. Custom scheduler gradually builds toward hot/cold architecture.

  • Phase 1 (quick): 80× AX162 with memory overcommit, cgroups limits, swap — €12,000/mo
  • Phase 2 (optimized): Hot/cold scheduler, reduces to 30 nodes — €4,500/mo
  • Why not cloud: At 50K, cloud costs are 10-30× higher; with 1 person + AI managing ops, the savings go straight to the bottom line

🔴 Tier 3: 500,000 Agents

10,000 concurrent (2%) · Custom orchestration mandatory · 29,500 vCPU peak

⚠️ 500K Requires Purpose-Built Infrastructure

At half a million agents, no off-the-shelf solution works. You need: (1) custom container lifecycle management, (2) tiered storage (NVMe hot → HDD warm → object cold), (3) geographic distribution, (4) dedicated networking. This is platform engineering, not just ops.

🏆 500K Recommendation: Hetzner Dedicated Fleet (Phase 1) → Hybrid (Phase 2)

Phase 1: €22,500/mo — Pure Hetzner with custom orchestrator. Build the hot/cold/frozen container lifecycle. With 1 person + AI managing operations, the infrastructure cost is 8-15× cheaper than cloud.

Phase 2: €85K-120K/mo — Once you hit geographic distribution needs or enterprise compliance requirements, add cloud regions (Azure/AWS) for specific markets while keeping Hetzner as the cost-efficient core.

  • Key engineering: Custom scheduler, container checkpointing (CRIU), tiered storage, geographic routing — AI agents automate most operational tasks
  • Alternative: If no engineer available, go Azure AKS at ~€180K/mo and trade $ for ops simplicity

Security Verdict

For enterprise/compliance-heavy customers: Azure AKS or AWS EKS offer the richest compliance certifications and managed security tools. For cost-optimized with good security: Hetzner + K8s with Cilium network policies, gVisor runtime sandboxing, and HashiCorp Vault provides strong isolation at a fraction of the cost. Hetzner DCs are ISO 27001 certified and GDPR compliant.

💡 The 1 Person + AI Model

With AI agents handling monitoring, auto-remediation, scaling decisions, log analysis, and routine ops tasks 24/7, one skilled engineer + AI replaces traditional 4-6 person SRE teams. The AI handles the toil; the human handles architecture decisions and edge cases. This makes Hetzner's cost advantage even more decisive — no need to factor in large team salaries.

🛤️ Migration Path

Phase 0: Current (Now — 60 agents)

  • 3× Hetzner AX42, manual Docker, custom provisioner
  • Cost: €150/mo
  • Keep as-is for alpha users

Phase 1: Beta (Next 3 months — 500 agents)

  • Add 2× AX162 servers, install K3s
  • Migrate provisioner to Helm chart deployments
  • Set up monitoring (Prometheus + Grafana)
  • Test namespace isolation, resource limits
  • Cost: ~€450/mo

Phase 2: Production 5K (Month 3-6)

  • Scale to 8× AX162 K3s cluster
  • Replace nginx tenant-map with K8s Ingress (Traefik)
  • Implement proper CI/CD pipeline
  • Add Cilium for network policies
  • Cost: ~€1,200/mo

Phase 3: Scale 50K (Month 6-12)

  • Migrate K3s → full K8s for better multi-node management
  • Scale to 80× AX162
  • Build hot/cold container scheduler (swap-based initially)
  • Add HashiCorp Vault for secrets
  • AI agents handle monitoring, auto-scaling, incident response
  • Cost: ~€12,000/mo

Phase 4: Scale 500K (Year 2+)

  • Custom orchestration layer with CRIU checkpointing
  • Tiered storage: NVMe → HDD → object storage
  • Geographic distribution (EU + US regions)
  • Consider hybrid cloud for enterprise compliance needs
  • 1 platform engineer + AI agents for ops automation
  • Cost: €22,500-120,000/mo depending on architecture

When to Choose Cloud Instead

  • No engineer available: If you have zero technical staff, managed cloud (AKS/GKE) trades money for ops simplicity
  • Enterprise sales: Some enterprises require SOC2/HIPAA cloud hosting — offer Azure/AWS as premium tier
  • Geographic expansion: Cloud regions are faster to deploy than colocating in new DCs
  • Burst capacity: Use cloud for overflow during usage spikes, Hetzner for steady-state
