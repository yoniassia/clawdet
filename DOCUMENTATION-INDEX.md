## Documentation Standards

All Clawdet documentation follows these standards:

### Structure
- Clear table of contents
- Progressive disclosure (basic ‚Üí advanced)
- Code examples for technical content
- Screenshots/diagrams where helpful

### Writing Style
- **Clear and concise** ‚Äî No jargon without explanation
- **Actionable** ‚Äî Tell users what to do, not just what exists
- **Examples** ‚Äî Show, don't just tell
- **Updated** ‚Äî Include last update date

### Maintenance
- Review documentation quarterly
- Update with new features
- Link related documents
- Archive outdated content
